The Go client library uses `option.WithEndpoint()` to redirect all API calls to fake-gcs-server instead of Google Cloud Storage. When the server runs over plain HTTP, no credential configuration is needed — the client does not attempt to fetch credentials for a non-HTTPS endpoint.

<Note>
  The example below requires the server to be started with `-public-host localhost:8080` so that resumable upload redirect URLs resolve correctly from the client.
</Note>

## Start the server

Run fake-gcs-server over plain HTTP on port 8080 with an in-memory backend:

```bash
docker run -d \
  --name fake-gcs-server \
  -p 8080:8080 \
  fsouza/fake-gcs-server \
  -scheme http \
  -port 8080 \
  -external-url http://localhost:8080 \
  -backend memory
```

Seed data is optional. Mount a local directory to pre-populate buckets and objects:

```bash
docker run -d \
  --name fake-gcs-server \
  -p 8080:8080 \
  -v "${PWD}/examples/data:/data" \
  fsouza/fake-gcs-server \
  -scheme http \
  -port 8080 \
  -external-url http://localhost:8080 \
  -backend memory
```

## Install the client library

Add the dependencies to your Go module:

```bash
go get cloud.google.com/go/storage
go get google.golang.org/api
```

## Configure the client

Point the client at `http://localhost:8080/storage/v1/` using `option.WithEndpoint`. When the server runs over plain HTTP you do not need to configure certificate validation:

```go
import (
    "context"

    "cloud.google.com/go/storage"
    "google.golang.org/api/option"
)

client, err := storage.NewClient(
    context.TODO(),
    option.WithEndpoint("http://localhost:8080/storage/v1/"),
)
```

<Warning>
  `option.WithoutAuthentication()` is **not** required when using `option.WithEndpoint` with an HTTP URL — the client will not attempt to fetch credentials for a non-HTTPS endpoint. If you run the server with HTTPS you must also disable TLS certificate verification or install the server's self-signed certificate.
</Warning>

## Complete example

The following program connects to fake-gcs-server, lists objects in a bucket, downloads a file, deletes it, and then updates the server's `externalUrl` configuration via the internal API:

```go main.go
// This example requires the server to be running with the flag -public-host
// defined as localhost:8080.
package main

import (
	"context"
	"fmt"
	"io"
	"log"
	"net/http"
	"strings"

	"cloud.google.com/go/storage"
	"google.golang.org/api/iterator"
	"google.golang.org/api/option"
)

func main() {
	// please notice that the sample server is running with http, if you
	// want to test this with https you also need to configure Go to skip
	// certificate validation.
	client, err := storage.NewClient(context.TODO(), option.WithEndpoint("http://localhost:8080/storage/v1/"))
	if err != nil {
		log.Fatalf("failed to create client: %v", err)
	}
	const (
		bucketName = "sample-bucket"
		fileKey    = "some_file.txt"
	)
	buckets, err := list(client, bucketName)
	if err != nil {
		log.Fatalf("failed to list: %v", err)
	}
	fmt.Printf("buckets: %+v\n", buckets)

	data, err := downloadFile(client, bucketName, fileKey)
	if err != nil {
		log.Fatal(err)
	}
	fmt.Printf("contents of %s/%s: %s\n", bucketName, fileKey, data)

	err = deleteFile(client, bucketName, fileKey)
	if err != nil {
		log.Fatal(err)
	}

	err = updateConfig()
	if err != nil {
		log.Fatal(err)
	}
}

func list(client *storage.Client, bucketName string) ([]string, error) {
	var objects []string
	it := client.Bucket(bucketName).Objects(context.Background(), &storage.Query{})
	for {
		oattrs, err := it.Next()
		if err == iterator.Done {
			break
		}
		if err != nil {
			return nil, err
		}
		objects = append(objects, oattrs.Name)
	}
	return objects, nil
}

func downloadFile(client *storage.Client, bucketName, fileKey string) ([]byte, error) {
	reader, err := client.Bucket(bucketName).Object(fileKey).NewReader(context.TODO())
	if err != nil {
		return nil, err
	}
	defer reader.Close()
	return io.ReadAll(reader)
}

func deleteFile(client *storage.Client, bucketName, fileKey string) error {
	return client.Bucket(bucketName).Object(fileKey).Delete(context.TODO())
}

func updateConfig() error {
	changeExternalUrl := "http://localhost:8080/_internal/config"

	client := &http.Client{}
	req, err := http.NewRequest(http.MethodPut, changeExternalUrl, strings.NewReader("{\"externalUrl\": \"http://localhost:8080\"}"))
	req.Header.Add("Content-Type", "application/json")
	if err != nil {
		return err
	}
	_, err = client.Do(req)

	return err
}
```

## How each operation works

<AccordionGroup>
  <Accordion title="Listing objects">
    `client.Bucket(bucketName).Objects()` returns an iterator. Call `it.Next()` in a loop and stop when it returns `iterator.Done`:

    ```go
    it := client.Bucket(bucketName).Objects(context.Background(), &storage.Query{})
    for {
        oattrs, err := it.Next()
        if err == iterator.Done {
            break
        }
        if err != nil {
            return nil, err
        }
        objects = append(objects, oattrs.Name)
    }
    ```
  </Accordion>
  <Accordion title="Downloading an object">
    Open a reader with `Object.NewReader`, then read all bytes:

    ```go
    reader, err := client.Bucket(bucketName).Object(fileKey).NewReader(context.TODO())
    if err != nil {
        return nil, err
    }
    defer reader.Close()
    return io.ReadAll(reader)
    ```
  </Accordion>
  <Accordion title="Updating the external URL">
    When fake-gcs-server is started without `-external-url`, you can set it at runtime via the internal API. This is required before making resumable uploads when the server URL is not known at startup (e.g. when using Testcontainers):

    ```go
    req, _ := http.NewRequest(
        http.MethodPut,
        "http://localhost:8080/_internal/config",
        strings.NewReader(`{"externalUrl": "http://localhost:8080"}`),
    )
    req.Header.Add("Content-Type", "application/json")
    http.DefaultClient.Do(req)
    ```
  </Accordion>
</AccordionGroup>
