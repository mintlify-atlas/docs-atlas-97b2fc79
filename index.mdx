fake-gcs-server provides a fully-featured emulator for the Google Cloud Storage API. Use it to run integration tests against a local GCS-compatible server without touching real GCP resources. It works with any language that supports the GCS API and also ships as a native Go library for in-process testing.

## What is fake-gcs-server?

fake-gcs-server emulates the Google Cloud Storage JSON API (`/storage/v1/`), the XML API, resumable uploads, multipart uploads, signed URLs, and Pub/Sub event notifications — all locally. Your existing GCS client libraries connect to it without code changes.

## Get started

<Steps>
  <Step title="Pull the Docker image">
    ```bash
    docker pull fsouza/fake-gcs-server
    ```
  </Step>
  <Step title="Start the server">
    ```bash
    docker run -d --name fake-gcs-server -p 4443:4443 fsouza/fake-gcs-server
    ```
  </Step>
  <Step title="Verify it's running">
    ```bash
    curl --insecure https://0.0.0.0:4443/storage/v1/b
    ```
    You should see `{"kind":"storage#buckets","items":[],"prefixes":null}`.
  </Step>
  <Step title="Point your client at the emulator">
    Configure your GCS client to use `https://0.0.0.0:4443` as the storage endpoint. See [Client Examples](/examples/go) for language-specific setup.
  </Step>
</Steps>
